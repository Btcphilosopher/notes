HTML6 Proposal: Vision and Research Agenda Executive Summary: HTML6 is a hypothetical next-generation web language, envisioned as an evolution of the living WHATWG HTML standard (HTML5+) rather than a single-version release. In practice, WHATWG continually extends HTML (e.g. adding , lazy-loading images, Shadow DOM, etc.) . For HTML6, proposals focus on richer component and module support, stronger formal foundations, and tighter integration with modern web concerns (security, privacy, AI). Key candidate features include an HTML Modules system (importing HTML as a module) , new semantic elements (e.g. , proposed , , for speech interfaces ), and built‑in metadata controls (e.g. new attributes or tags to express privacy preferences or security policies). Formal foundations from programming-language theory (type systems, formal semantics, effect systems) are largely absent in current HTML, but could be added in HTML6 for verification or security (e.g. statically checking an fits its min/max). Integration with advanced web APIs is expected: for example, W3C’s GPU-for-Web work is finalizing WebGPU (for GPU-accelerated graphics/compute) , and the Web Neural Network API (WebNN) provides a hardware-agnostic layer for on-device AI . HTML6 could surface these through new tags or declarative attributes. We examine these domains in detail below, with examples, trade-offs, and references. 1. Proposed Language Features 1.1 Syntax and New Elements HTML6 would continue HTML’s HTML5+ syntax (permissive parsing, optional end-tags, etc.), ensuring backward compatibility. The WHATWG FAQ emphasises that the spec must recover from all invalid markup (“tag soup”) to interoperate with existing pages . Thus HTML6 must retain HTML5-style parsing rules (e.g. allowing trailing slashes on void tags for XHTML compatibility ). Beyond that, HTML6 proposals include new tags and attributes for richer semantics: Modal/dialog elements: HTML5.2 added for native modal windows . Browsers now implement it (Chrome, Firefox, Safari support it since ~2022 ). HTML6 could extend this concept with additional dialog types or default ARIA roles. The spec defines boolean attributes like open (to control visibility) and enums like closedby , and emphasises author guidance (e.g. use autofocus) .
